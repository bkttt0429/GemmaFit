# GemmaFit v5 E2B Data Source Mapping

This file maps public or app-generated data sources to the v5 E2B evidence
router. The v5 dataset trains function routing, evidence citation, persona
wording, and refusal behavior. It does not train medical conclusions or
single-camera force/GRF/EMG claims.

## Allowed Use

```text
public video/skeleton/mocap source
-> derived compact evidence rows
-> capability contract / refusal examples
-> v5 evidence-to-function SFT rows
```

Raw video, raw skeleton sequences, force plates, GRF, EMG, clinical labels,
diagnoses, fall-risk scores, sarcopenia labels, and rehab outcomes are not
direct SFT targets.

## Conversion Rules

- Convert source examples into compact evidence packets before SFT.
- Preserve source provenance in metadata, not user-facing model output.
- Use blocked capability nodes for unsupported metrics.
- Use `tracking_state=predicted/lost/multi_person_ambiguous` rows to train
  abstention and safe fallback behavior.
- Use subjective effort rows only when the source is explicitly self-report or
  synthetic bounded check-in.
- Keep all sensor-only concepts as `cannot_judge` unless GemmaFit actually has
  that sensor evidence.

## v5.1 Hard-case Expansion

The v5.1 dataset is generated by the same script with `--hard-cases`. It is
still an evidence-router SFT dataset, not a raw video or clinical-label
dataset.

Recommended main-training command:

```powershell
python finetune/data/generate_v5_e2b_evidence_router.py `
  --hard-cases `
  --train-size 50000 `
  --validation-size 6000 `
  --zh-tw-ratio 0.45 `
  --schema-fuzz-ratio 0.25 `
  --output finetune/data/gemmafit_v5_1_e2b_evidence_router.json `
  --validate
```

Added row families:

| Dataset family | Purpose | Typical source |
| --- | --- | --- |
| `schema_fuzz` | Prevent overfitting to exact field order and optional-field presence. | Synthetic packet perturbation. |
| `tracking_uncertainty` | Block hard judgment for predicted, lost, ambiguous, or low-confidence subjects. | Synthetic + Pixel debug failures. |
| `parent_task_uncertain` | Prevent parent-task guesses from enabling sport-specific hard rules. | Synthetic + Layer 2 uncertainty cases. |
| `sub_action_fallback` | Use high-confidence sub-actions when parent task is uncertain. | BABEL/FineGym-style phase labels after conversion. |
| `conflicting_evidence` | Cite only non-conflicting evidence and downgrade blocked metrics. | App debug packets and synthetic contradictions. |
| `memory_policy` | Require evidence ids for care-log and dual-task memory writes. | Synthetic + app audit logs. |
| `unsupported_zh_tw` | Train Traditional Chinese refusal wording without medical or force claims. | Synthetic boundary prompts. |

Public datasets help v5.1 only after conversion:

```text
public source -> pose/phase/event features -> compact evidence packet
-> capability_contract -> expected function-call row
```
